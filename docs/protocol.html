EvidenceMap: A Browser-Based Evidence Gap Map Generator — Protocol

A free, open-source tool for generating interactive evidence gap maps with GRADE-colored bubble charts, heatmaps, and automated gap priority analysis — requiring no software installation.

Abstract. Evidence gap maps (EGMs) provide a visual synthesis of what is known and unknown across a predefined intervention×outcome matrix. Existing EGM tools are either proprietary (EPPI-Mapper, 3ie Gap Map Tool) or require specialist statistical software (R, Stata). We describe the development protocol for EvidenceMap — a single-file HTML application that generates interactive EGMs entirely in the browser, with no server dependency, no installation requirement, and no cost. Primary outputs include GRADE-colored bubble charts (size proportional to study count k), heatmaps, and ranked gap priority tables. The tool supports manual data entry and CSV import, with PNG/HTML/CSV export.

1. Background & Rationale

Evidence gap maps were first formalized by the International Initiative for Impact Evaluation (3ie) as a structured approach to visualizing the breadth and depth of evidence across intervention domains. Unlike traditional systematic reviews that answer a focused question, EGMs map the entire landscape of evidence — identifying where high-certainty evidence exists, where evidence is sparse or absent, and where future research should be prioritized.

Despite methodological advances, practical barriers to EGM production remain high. The dominant tool (EPPI-Mapper) requires institutional access. R-based alternatives (package egm) assume programming literacy. 3ie's own gap map repository accepts curated submissions but does not provide a self-service authoring tool. This creates a systematic barrier for researchers in low- and middle-income countries and for independent investigators.

"The most evidence-poor regions are also the regions least able to afford the tools to document that poverty."

A browser-based solution addresses this gap directly. Modern browsers support Canvas 2D, SVG, File API, and localStorage — sufficient to implement a full EGM authoring workflow without any backend. The resulting tool can be shared as a single HTML file, hosted on GitHub Pages, or run offline.

2. Objectives

The primary objective is to build a free, open-access browser tool that enables any researcher to:

3. Methods

Architecture. EvidenceMap is implemented as a single self-contained HTML file (<3,000 lines). The visualization layer uses the HTML5 Canvas 2D API for bubble charts and heatmaps, avoiding external library dependencies. UI state is managed in a plain JavaScript object (appState) serialized to localStorage with key evidenceMap_v1.

Intervention×outcome grid. Users define a list of interventions (rows) and outcomes (columns). Each cell in the matrix stores: study count k, pooled effect size (free-text), certainty of evidence (GRADE four-level scale: High/Moderate/Low/Very Low/Not assessed), and direction of effect (Beneficial/Harmful/No effect/Mixed/Unknown).

Bubble chart rendering. Bubble radius is computed as r = MIN_R + (MAX_R − MIN_R) × sqrt(k / k_max), mapping study count onto area proportionally. GRADE certainty determines fill color via a fixed palette: High (#276749 green), Moderate (#2b6cb0 blue), Low (#c05621 orange), Very Low (#c53030 red), Not assessed (#718096 grey). Direction arrows overlay each bubble.

Heatmap rendering. Cell intensity is proportional to k using a blue sequential color scale (white = 0 studies; deep blue = maximum k observed). Zero-study cells are cross-hatched to distinguish "not assessed" from "assessed, k=0".

Gap priority analysis. Each intervention×outcome cell is scored:

Priority = 0.4×(1 − certainty_norm) + 0.4×(1 − k_norm) + 0.2×(1 − direction_norm)

where certainty_norm maps GRADE levels to [0, 1], k_norm = k / k_max, and direction_norm = 0 for Unknown/Not assessed, 0.5 for Mixed, 1 for known direction. Cells with k=0 and certainty "Not assessed" receive maximum priority (1.0).

CSV import/export. CSV import parses the first row as a header. Required columns: intervention, outcome, k; optional: effect, certainty, direction. Export serializes all cells in the same schema.

Zoom and filter controls. Users can zoom the bubble map (50%–200%) and filter by minimum k, certainty level, or direction. Filters apply in real time without re-rendering from scratch.

5. Primary Outcomes

Bubble map. Interactive Canvas rendering of the full intervention×outcome matrix. Hovering a bubble triggers a tooltip with cell details. Zoom control adjusts the viewport.

Heatmap. Color-density grid using blue sequential scale. Suitable for rapid visual triage of evidence-dense vs. evidence-poor cells.

Gap priority table. Ranked list of intervention×outcome pairs by computed priority score (0–100). Columns: rank, intervention, outcome, k, certainty, direction, priority score.

Export functions. PNG export uses canvas.toDataURL(). HTML export embeds the full rendered page. CSV export serializes the current matrix to RFC 4180 format.

6. Limitations

The current protocol does not include automated search functionality (PubMed/Embase API integration) — data entry is manual or via pre-processed CSV. The tool does not enforce a specific EGM methodology (Campbell, 3ie, or EPPI) and makes no attempt to replicate proprietary scoring systems. Statistical pooling is outside scope; effect size fields are free-text annotations only.
