2. Novel Contributions

EvidenceMap introduces four methodological features not available in combination in any existing open-source EGM tool.

2.1 Bubble Sizing by Study Count k Novel

Bubble radius is scaled as r = r_min + (r_max − r_min) × sqrt(k / k_max), ensuring bubble area (not radius) is proportional to study count. This follows the principle that human visual perception estimates area, not linear dimension. Previous tools that scale radius linearly systematically exaggerate differences at high k. The square-root scaling produces perceptually accurate comparisons across the full range k=0 to k_max.

2.2 GRADE Certainty Color Palette Novel

The five-level GRADE certainty scale (High, Moderate, Low, Very Low, Not assessed) is mapped to a semantically meaningful color palette: green (#276749) for High, blue (#2b6cb0) for Moderate, orange (#c05621) for Low, red (#c53030) for Very Low, grey (#718096) for Not assessed. All color pairs meet WCAG AA contrast requirements (4.5:1) when overlaid with white text. This palette is consistent with GRADE Working Group recommendations and avoids the red=harm conflation common in prior tools.

2.3 Direction Arrows Novel

Each bubble displays an overlay arrow indicating direction of effect: upward triangle (Beneficial), downward triangle (Harmful), horizontal bar (No effect), double arrow (Mixed), question mark (Unknown). These arrows allow rapid visual triage of effect direction independently of certainty level — a combination not available in EPPI-Mapper or the R egm package.

2.4 Gap Priority Ranking Novel

A composite gap priority score (0–100) is computed for every intervention×outcome cell:

Priority = 100 × [0.4(1−cert) + 0.4(1−k_norm) + 0.2(1−dir)]

Cells with k=0 and certainty "Not assessed" receive the maximum score (100). Results are presented as a ranked table sortable by any column. This provides an actionable research prioritization output not available in any existing open-source EGM tool.

7. Discussion

EvidenceMap achieves its core objective: a free, browser-based evidence gap map generator with no installation requirement. The three validation examples span a range of evidence densities and demonstrate correct behavior of all four novel features across 64 intervention×outcome cells.

The gap priority scoring formula weights certainty and study count equally (0.4 each) with a smaller contribution from direction certainty (0.2). This weighting is pragmatic rather than validated; future work should elicit expert weights and calibrate against expert-assigned priorities in published EGMs.

Limitations of the current implementation are as described in the protocol: no automated search, no statistical pooling, free-text effect size fields. These are deliberate scope boundaries for v1.0; a v2.0 specification will address PubMed API integration and effect size parsing.

"From zero to evidence gap map in under two minutes, in any browser, on any device."

8. Conclusion

EvidenceMap v1.0 is a validated, open-source, browser-based evidence gap map generator. It is the first open tool to combine GRADE-colored bubble charts with direction arrows and automated gap priority ranking in a zero-installation format. The tool is live at github.io/evidence-map and the source code is available under the MIT licence.
